# My-Work-Exercise-Bright-Tv-Case-Study / BrightTV Viewership Analysis – Case Study

## Overview
This repository contains the analysis for the BrightTV Viewership Case Study (2026). The objective was to provide insights to help BrightTV grow its subscription base by analyzing user behavior, viewing trends, and content preferences.

The analysis includes:
- User and usage trends
- Factors influencing consumption
- Content recommendations for low-consumption days
- Initiatives to grow BrightTV’s user base

All insights are based on the **viewership** and **user profile** datasets provided.

## Key Insights

### User & Usage Trends
- Peak viewing days: **Friday and Saturday**  
- Lowest viewing day: **Monday**  
- Peak hours: **17:00–19:00**  
- Top content: **Supersport Live Events**, **ICC Cricket World Cup 2011**, **Channel O**  
- Majority of users: **Young adults (26–39 years), mostly male, concentrated in Gauteng, Western Cape, and KwaZulu-Natal**

### Factors Influencing Consumption
- Age: Young adults drive most engagement 
- Gender: Males dominate sports content  
- Province: Urban areas contribute most sessions  
- Time: Evenings see the highest activity  

### Content Recommendations for Low-Consumption Day (Monday)
- Schedule popular sports content and trending entertainment channels  
- Include children’s content in the afternoon  
- Use push notifications and promotions to increase engagement  

### Initiatives to Grow BrightTV User Base
- Launch exclusive content and curated playlists  
- Referral programs and targeted marketing campaigns  
- Monday-specific promotions and trial subscriptions  
- Partner with mobile/data providers for bundled subscriptions  
- Improve platform UX, add loyalty rewards, and kid-friendly content



## How to Run the Notebook

1. Open `BrightTV_Viewership_Analysis.ipynb` in Databricks or Jupyter Notebook.  
2. Ensure datasets (`user_profile.csv` and `viewership.csv`) are available in the working directory or DBFS.  
3. Run the notebook **from top to bottom** to reproduce all charts and analysis.  
4. Add visualizations to a Databricks dashboard if desired.
